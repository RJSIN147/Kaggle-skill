<objective>
Make SCORE-02 real: CV remains the decision metric everywhere, and the framework computes and trends
the CV-to-LB gap per experiment and raises a divergence alarm when CV and LB disagree.

The alarm fires on RANK INVERSION (D-10): experiment B has a better CV than A, but scores worse on the
leaderboard. This was chosen over an absolute |CV - LB| gap threshold because a large but STABLE offset
between CV and LB is usually benign (different data, different split) - what actually breaks a
competition run is CV ORDERING ceasing to predict LB ordering. Rank inversion is scale-free (identical
for AUC, RMSE, LogLoss with no per-competition tuning) and it catches the failure that matters: CV has
stopped being a trustworthy decision metric.

The gap itself is still computed and trended per experiment (criterion 2 requires the trend); rank
inversion is what raises the ALARM.

The view is DERIVED (D-11): join `submissions.jsonl` to `ledger.jsonl` on `exp_id`. The LB score is
never written back into `meta.json` - that would mutate a folder Phase 3 treats as immutable after
record and would create two sources of truth to keep in sync. The join also naturally handles
many-submissions-per-experiment, which a single meta.json field cannot.
Output: `scripts/lb_gap.py` + an extension (never a fork) of the Phase 3 strategy-regen contract.
</objective>

    join_cv_lb(sub_rows, ledger_rows) -> list[dict] (D-11, SCORE-02):
      - Index ledger_rows by exp_id, keeping only status == "SUCCESS" rows with a numeric cv_mean.
      - For each submissions row with status == "SCORED" and a NON-None public_score, and whose exp_id
        matches a ledger row, emit {"exp_id", "cv_mean", "cv_std", "lb_score", "gap", "kaggle_ref",

        exp_id is null (an out-of-band submission back-filled by fetch_lb --reconcile) or absent from
        the ledger. Never fabricate a score.
      - Sort by scored_at (or submitted_at) ascending - this ordering IS the trend.
      - Naturally handles MANY submissions per experiment: each scored submission is its own row.

    rank_inversions(pairs, greater_is_better) -> list[tuple] (D-10):
      - pairs: [(exp_id, cv_mean, lb_score)] for SCORED submissions only.
      - better(x, y) = x > y if greater_is_better else x < y  # direction-aware; the SAME direction
        applies to both CV and LB, because Kaggle reports publicScore in the competition's own metric.
      - For every unordered pair (a, b) via itertools.combinations: an INVERSION exists when
        better(b_cv, a_cv) AND better(a_lb, b_lb) - CV says B wins, the LEADERBOARD says A wins.
      - Return the inversions with the CV delta and the LB delta so the renderer can name the numbers.
      - SCALE-FREE: no threshold, no per-competition tuning. Works identically for AUC, RMSE, LogLoss.
      - Requires >= 2 scored points to fire. With 0 or 1, return [] - and be HONEST about it (see the
        renderer below); never fake a signal from a single point.

    alarm_state(pairs, greater_is_better) -> dict:
      - {"n_scored": int, "can_fire": bool, "inversions": [...]}. can_fire is False when n_scored < 2.
      - This is what lets the renderer print the honest "Divergence alarm: needs >=2 scored submissions
        (have N)." line instead of a fabricated all-clear.
