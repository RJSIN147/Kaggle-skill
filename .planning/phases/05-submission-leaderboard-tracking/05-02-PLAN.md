<objective>
Close the cross-phase gap D-09 identified: Phase 3's `run_cv` harness cross-validates and writes
`result.json`, but it never fits-and-predicts on test - so **`submission.csv` does not exist
anywhere in this codebase today**. Without this plan there is literally nothing for SCORE-01 to
submit.

Mechanism (D-09, locked): reuse the CV fold models. Each fold's model and its fitted preprocessor
already exist inside the loop; predict test with each and aggregate. This is free (no second
training run - real GPU time on the kernel path), is a mild ensemble, and every contributing model
was actually CV-scored.

Purpose: make `submission.csv` a first-class hashed experiment artifact that flows back through the
Phase 4 `pull_kernel` path with zero changes to `pull_kernel.py`, and through `record_experiment.py`
with zero recorder changes.
Output: an extended `run_cv` harness + a scaffolder that bakes the submission header in.
</objective>

    BEFORE the fold loop: build Xte_v by applying feature_fn to X_test when feature_fn is set, then

    INSIDE the fold loop, immediately AFTER model.fit(Xtr, ytr) and reusing THAT SAME fold's fitted

      - ptype == "label" and hasattr(model, "predict_proba"): append model.predict_proba(Xte) to
        test_probas AND model.predict(Xte) to test_preds (the majority-vote fallback material).
      - otherwise (raw regression, or a label model with no predict_proba): append
        model.predict(Xte) to test_preds.
    Capture the last fold's model.classes_ (via getattr(model, "classes_", None)) for the argmax.

    AFTER the fold loop, add a module-level helper _default_agg(test_preds, ptype, test_probas,
    classes) implementing the TYPE-AWARE aggregation:
      - ptype in ("proba", "raw"): np.mean(test_preds, axis=0). Correct - a soft ensemble.
      - ptype == "label": if test_probas is non-empty AND classes is not None, take
        np.mean(test_probas, axis=0) then classes[np.argmax(..., axis=1)] (soft-vote then argmax -
        a calibrated soft ensemble). Otherwise fall back to a per-row MAJORITY VOTE over
        np.vstack(test_preds) using collections.Counter.
      ⚠ NEVER np.mean over hard labels. Averaging five folds of 0/1 yields 0.6, which is not a
      member of the label set; D-02's validator (headers/count/ids/blanks) would PASS that garbage

      path.
