<objective>
Build the two entry points that touch the irreversible surface: `submit.py` (spends the slot) and
`fetch_lb.py` (the detach fallback that records the score later).

The single most important correctness constraint in this phase lives here. **`kaggle competitions
submit` is FAIL-OPEN: it prints a message and exits 0 on a 404 slug and on a failed upload.** The
client swallows its own 404 before it can propagate, and a failed upload returns a message object
rather than an error. `rc == 0` therefore does NOT prove the submission landed. Success must be
confirmed by READ-BACK - which conveniently is also the only channel that yields the Kaggle `ref` id
(the CLI discards it) and doubles as the first poll tick. This is structurally identical to Phase 4's
"a kernel can report COMPLETE and still have lied" - reuse the instinct.

Output: `scripts/submit.py`, `scripts/fetch_lb.py`, and a backward-compatible widening of
`poll_kernel.compute_delay` so the LB loop reuses the already-unit-tested backoff math.
</objective>
