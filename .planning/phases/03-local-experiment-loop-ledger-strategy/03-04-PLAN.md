<objective>
Deliver the integrity spine of the phase — the anti-lie guarantee. `scripts/run_local.py` runs
the scaffolded `experiment.py` under `uv run --no-sync`, capturing ONLY the exit code (never
scraping stdout for a score). `scripts/record_experiment.py` then applies the D-06 fail-closed
validation ladder to the on-disk `result.json`, recomputes `mean(fold_scores)` to catch a lying
notebook, attaches stdlib provenance (run id, artifact hash, git commit, seed), and persists the
numbers into `meta.json` + a derived `ledger.jsonl` row + a `VERDICT.md` stub. A run that throws,
is missing/schema-invalid, non-finite, out-of-range, or whose mean disagrees is recorded as a
FAILURE **with a verdict** — never upgraded to success.

Purpose: this is criterion 3 — "numeric fields written only by tooling; a deliberately-throwing
notebook is recorded as a failure, not a success; every row carries provenance" (EXP-03, EXP-04,
D-05, D-06). It is the MVP headline slice: after this plan a real user can watch a throwing
experiment be caught and logged honestly.
Output: `scripts/run_local.py`, `scripts/record_experiment.py`, and their Wave-0 tests including
the end-to-end anti-lie test.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| experiment.py emitted result.json → recorder | The emitted numbers are untrusted; only the recorder's validated output becomes a score |
| run_local exit code / stdout → recorder | stdout is never trusted for a score; only the on-disk result.json + exit code drive classification |
| recorder → git provenance | Provenance must be honest (never claim a clean commit it did not have) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-04-01 | Tampering | fabricated/hand-written score | mitigate | numbers written ONLY by the recorder from a schema-validated result.json; AI never hand-writes (EXP-04); recorder recomputes mean(fold_scores), rejects any mismatch |
| T-03-04-02 | Spoofing | silent failure recorded as success | mitigate | D-06 fail-closed ladder: nonzero exit OR missing OR schema-invalid OR non-finite OR out-of-range → FAILED-with-verdict, no success row (criterion 3) |
| T-03-04-03 | Repudiation | non-reproducible provenance | mitigate | run_id (uuid4), artifact_hash (sha256 of experiment.py), git_commit (rev-parse) + git_dirty flag, seed — on every row (EXP-04) |
| T-03-04-04 | Information disclosure | git staging sweeps secrets | mitigate | stage experiment.py + meta.json by EXPLICIT path (never `git add -A`; the leak guard trips on control/raw/last-error.txt) |
| T-03-04-05 | Elevation of privilege | experiment.py code execution | mitigate | `uv run --no-sync` (no network install), timeout-bounded, inherits the workspace egress sandbox (no internet by default) |
</threat_model>
