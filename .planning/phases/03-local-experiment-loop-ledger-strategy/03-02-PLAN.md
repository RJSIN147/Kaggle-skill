<objective>
Establish the git-backed ledger contract: an `experiment_meta.py` stdlib module that owns the
`meta.json` ⇄ `ledger.jsonl` row schema in ONE place (`to_ledger_row` + `validate_meta`), a
`rebuild_ledger.py` entry point that reconstructs `ledger.jsonl` as a pure function of the
per-experiment folders, and the `meta.json` / `VERDICT.md` templates. This is the MEM-01
canonical/derived pair and the provenance backbone (EXP-04) that the recorder (03-04) writes into.

Purpose: D-10 makes `meta.json` canonical and `ledger.jsonl` fully rebuildable, so a
hand-corrupted ledger self-heals and every row carries auditable provenance. Defining the schema
here (before the recorder consumes it) means 03-04 implements against a fixed contract, not a
moving target.
Output: `scripts/experiment_meta.py`, `scripts/rebuild_ledger.py`, two templates, and Wave-0 tests.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| per-experiment meta.json → derived ledger.jsonl | Provenance must survive derivation intact and never be fabricated |
| corrupt/hand-edited meta.json → rebuild | A tampered folder must not silently produce a plausible row |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-02-01 | Tampering | provenance fields | mitigate | to_ledger_row copies run_id/artifact_hash/git_commit/seed straight from meta.provenance; validate_meta requires all four present before a row is emitted |
| T-03-02-02 | Repudiation | corrupt/partial meta.json | mitigate | rebuild skips-and-warns (stderr names the folder), never fabricates a row; no partial live-file write (atomic os.replace) |
| T-03-02-03 | Tampering | hand-corrupted ledger.jsonl | mitigate | ledger is a pure function of meta folders — a full rebuild self-heals it (MEM-01) |
| T-03-02-04 | Information disclosure | git staging of provenance | mitigate | (enforced in 03-04) stage meta.json by explicit path, never `git add -A` |
</threat_model>
