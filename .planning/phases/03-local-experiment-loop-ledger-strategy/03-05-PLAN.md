<objective>
Close the loop: `scripts/regen_strategy.py` regenerates `strategy.md` from `control/ledger.jsonl`
each cycle — tooling renders the FACTS (current-best by the metric's direction + the full
tried-list digest) and splices an AI-authored REASONING block (hypothesis queue + next action)
delivered via `--reasoning-file`, then overwrites the file WHOLESALE and atomically (D-12,
deliberately unlike competition.md's section-safe-merge). Then wire the whole
`scaffold -> run -> record -> regen` loop into `SKILL.md`, including the prompt-driven never-repeat
check the AI performs against the tried-list digest before authoring a new `experiment.py`.

Purpose: MEM-03 wants a living strategy regenerated from the ledger, never hand-edited, so a
stale edit can never drift from the facts; MEM-02 wants the AI to reason over history so it never
re-proposes a tried idea (D-11/D-12/D-13). This plan makes the cycle a documented, sequenced whole.
Output: `scripts/regen_strategy.py`, an updated strategy template header, the SKILL.md loop
section + Scripts rows, and the Wave-0 test.
</objective>
