<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| AI-supplied preprocess/model → run_cv fold loop | Untrusted-ish AI code plugged into the harness must not be able to leak val-fold data into training |
| generated experiment.py → uv run execution | The scaffolded script is code that will be executed locally |
| scripts/metric_registry.py → rendered experiment.py | The metric contract must stay a single source of truth; the experiment gets a rendered snapshot, never a drifting hand-copy |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-03-01 | Tampering | CV fold preprocessing (leakage) | mitigate | run_cv owns fit_transform(train)/transform(val); AI supplies an UNFITTED preprocess_factory; harness never accepts a fitted transformer or two-frame preprocess_fn (criterion 1, D-07) |
| T-03-03-02 | Elevation of privilege | generated experiment.py executed via uv run | mitigate | execution is bounded by run_local.py (03-04): `uv run --no-sync` (no network install), timeout, inherits the workspace egress sandbox (no internet by default); no network path in the harness |
| T-03-03-03 | Spoofing | wrong metric mapped / wrong prediction-type | mitigate | scorer resolved from the rendered `registry_entry["sklearn_callable"]` (never `getattr` on the config name); prediction_type drives predict vs predict_proba (Pitfall 3); multiclass needs a custom metric callable (first-class, Pitfall 4) |
| T-03-03-04 | Repudiation | duplicate/overwritten exp ids | mitigate | zero-padded cursor from state.next_exp_id; create_if_absent never clobbers a prior experiment.py (D-02 idempotency) |
| T-03-03-05 | Tampering | metric contract drift between skill and experiment | mitigate | single source of truth = scripts/metric_registry.py; the scaffolder renders a snapshot per experiment — no hand-maintained duplicate to drift (WARNING 2) |
</threat_model>
