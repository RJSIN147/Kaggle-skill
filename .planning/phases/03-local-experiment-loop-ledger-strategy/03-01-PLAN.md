<objective>
Build the metric foundation the whole result contract stands on: a stdlib `metric_registry.py`
(the single source of truth mapping a small enum of Kaggle metric names to direction, valid
range, prediction type, and the sklearn callable name), a `set_metric.py` setter that mirrors
the Phase-2 "AI decides / tooling writes" flow to persist `config.json.metric`, the reserved-null
`metric` key in the config template, and the ML dependency floors in the workspace pyproject so
`uv sync` can materialize the local run env.

Purpose: D-08 requires ONE machine source of truth read by the CV harness (metric callable),
the recorder (direction + valid-range gate), and strategy regen (current-best selection). This
plan produces that source and makes the metric enum-written, never AI-free-typed.
Output: `scripts/metric_registry.py`, `scripts/set_metric.py`, extended config + pyproject
templates, and their Wave-0 tests.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| AI reasoning → config.json.metric | The AI's metric judgment must not become a free-typed numeric-affecting value |
| workspace pyproject → operator `uv sync` | Declared deps installed by the operator; no runtime install by the skill |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01-01 | Tampering | metric write | mitigate | metric enum is argparse `choices=SUPPORTED`; direction looked up from REGISTRY, never free-typed; custom requires explicit direction or blocks |
| T-03-01-02 | Spoofing | wrong metric mapped to "close" scorer | mitigate | unmappable → `custom` + explicit direction or reserved-exit block; NEVER auto-map (RESEARCH §Anti-Patterns) |
| T-03-01-03 | Denial of service | corrupt config.json | accept/mitigate | fail-clear: corrupt JSON left byte-intact, non-zero exit (MalformedControlJSON posture) |
| T-03-01-SC | Tampering | workspace uv sync (lightgbm/xgboost/catboost/numpy) | accept | Package Legitimacy Audit (03-RESEARCH.md) — all household ML packages, disposition Approved (CITED); no [ASSUMED]/[SUS]; installed into the WORKSPACE env by the operator, not the skill; degrade-don't-install posture |
</threat_model>
