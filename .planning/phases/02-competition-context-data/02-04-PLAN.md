<objective>
Deliver the analysis slice: from the downloaded CSVs, emit structural CV evidence, let the AI commit
to a scheme (tooling writes the enum AND the human-readable rationale), and produce an
adversarial-validation finding â€” completing `competition.md` so success criterion 1 holds end to end.

Purpose: CV scheme is a structural fact, so it follows the same "tooling writes, AI never hand-writes"
posture Phase 3 applies to numeric results, applied one phase early (D-05). D-05 is a TWO-part
deliverable: the enum goes to `config.json.cv.scheme` AND a written rationale goes to `competition.md`'s
`## Cross-validation scheme` section â€” the doc a Phase 3 agent reads every cycle. The enum WRITE uses the
generalized `set_config_field` direct-overwrite setter from 02-02 â€” `write_control_json` (merge-add-missing)
would silently skip the reserved-`null` `cv.scheme` and leave it null forever. Real adversarial validation
(LogisticRegression on train=0/test=1, roc_auc_score) catches JOINT distribution shift â€” the failure mode
that destroys CVâ†’LB correlation (the exact quantity SCORE-02 tracks). Per-column marginal shift is a weaker
artifact; if only that runs, competition.md must say SKIPPED, not imply AV ran (D-06).

Output: `scripts/cv_evidence.py`; `scripts/analyze_data.py`; the ML floors in `pyproject.toml.tmpl`;
the CV-evidence test + a single fixture-builder helper generating grouped/temporal/imbalanced CSV pairs.
