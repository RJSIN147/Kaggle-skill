<objective>
Deliver the first operator-facing slice: capturing the competition "constitution" (metric, rules,
daily submission limit + provenance, competition type) into `competition.md`, with every ingested
Kaggle text passed through the untrusted-content boundary. This is where competition prose ENTERS
the system, so this plan owns the two mechanical, unit-testable guarantees that make success
criterion 2 real: the fence cannot be broken from inside, and no executed path/command/URL is
derived from competition text.

Purpose: `competition.md` is re-read into agent context on every experiment cycle from Phase 3
onward. A payload embedded there is re-read forever as trusted project doc (D-01). And Phase 5
rations a scarce budget against `submission.daily_limit` — a fabricated `5` is byte-identical to an
extracted `5` unless it carries provenance (D-13). Competition prose is ingested through the kaggle
CLI (`competitions pages`/`files`) rather than any web scraping — the CLI stays the sole primitive
(D-15), which is exactly why the untrusted-content boundary (D-01/D-02) is the sharp edge here.

Output: `scripts/untrusted.py` (escape_markers + fence writer); `scripts/capture_competition.py`;
the filled `competition.md.tmpl`; the extended `config.json.tmpl`; and the two named C2 deliverable
tests plus the limit-regex + capture tests.
</objective>
