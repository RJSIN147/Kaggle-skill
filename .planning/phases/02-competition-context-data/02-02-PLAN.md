CONFIG-FIELD COMMIT — the value-WRITE mechanism (BLOCKER: `write_control_json` CANNOT write these
fields). `write_control_json` → `deep_merge_add_missing` only ADDS keys that are ABSENT

value (e.g. `cv.scheme`) survives"). Because `config.json.tmpl` ships `cv.scheme` as `null` (Phase 1)
and this plan adds `submission`/`competition` as `null` too, every scaffolded workspace has all three
keys PRESENT-as-null before capture runs → `write_control_json({"submission":{"daily_limit":10}})`
returns `"skip"` and writes nothing. So:
```python
# NEW in scripts/init_workspace.py — GENERALIZE set_execution_target (do NOT fork):

    # malformed JSON → MalformedControlJSON posture (bytes preserved, non-zero). Returns 0 on success.

    return set_config_field(config_path, ("execution_target",), target)
```

  `competition`/`submission` as `null`) onto a pre-Phase-2 workspace — its documented structural use,
