<objective>
Deliver the download slice: clear the UI-only rules gate (or fail closed without ever busy-looping),
then download the competition data and extract it with zip-slip protection so no file can escape
`data/`. CLI 2.2.3 has NO `--unzip` flag (VERIFIED) — it pulls a single `<slug>.zip` — so manual
extraction is mandatory, which is exactly why criterion 4 demands a reject-and-raise guard.

Purpose: the gate flow must honor criterion 3 (never busy-loops): a cheap preflight probe classifies
the rules gate, prints the exact URL, and exits the reserved code — the SKILL holds the human loop and
the re-invocation's preflight IS the verification (D-10). And stdlib `zipfile.extractall` has no
`filter=` and SILENTLY drops `..`/absolute members, so the guard must reject-and-raise for the
malicious-archive test to assert refusal (criterion 4).

Output: `scripts/safe_extract.py`; `scripts/download_data.py`; the malicious-archive + gate tests.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| downloaded archive → data/ filesystem | Archive members are attacker-controllable; extraction is the escape surface |
| Kaggle 403 gate → human loop | The gate is UI-only state on kaggle.com; the framework must never busy-loop an authenticated endpoint |
| credential state → download op | Data download is a credential-dependent op gated on state.json |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-PATH-01 | Tampering / EoP | safe_extract | mitigate | Reject-and-raise on absolute/`..`/symlink members + realpath containment; validate all members before any write → test_no_file_escapes |
| T-02-DOS-01 | Denial of Service | download_data gate preflight | mitigate | Single cheap preflight; exit UI_GATE on gate; NO poll/backoff/input/sleep; re-invocation is the verification → test_gate asserts probe-once + sleep-not-called |
| T-02-GUESS-01 | Repudiation | classify_gate on generic 403 | mitigate | Fail closed — name BOTH rules + phone URLs, never pattern-match "phone" into the generic 403 |
| T-02-LEAK-01 | Information Disclosure | download error handling | mitigate | Raw CLI output → gitignored last-error.txt (dump_last_error); never echoed; never `git add control/raw/` |
| T-02-AUTHZ-01 | Elevation of Privilege | credential gate | mitigate | Refuse download unless state.json.credentials == VALIDATED (Phase 1 D-07); fail-clear on corrupt state |
| T-02-SC | Tampering | package installs | accept | No new skill packages (stdlib zipfile only) |
</threat_model>
