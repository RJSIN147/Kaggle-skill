<objective>
Close Gap 1 from `02-VERIFICATION.md`: the as-built analyze flow auto-commits the mechanical
`recommend_cv()` default, so a live run against the real `titanic` dataset commits
`cv.scheme = "GroupKFold"` (rationalized as "group leakage" on the continuous columns
`Age`/`Fare`/`Cabin`) instead of the textbook-correct `StratifiedKFold`. The root cause is
altitude, not just the detector — the framework picks the value at all, violating D-05
("Tooling recommends → AI reasons → tooling writes").

This plan re-wires the flow so the framework surfaces evidence + a non-authoritative advisory
hint, the AI decides, and tooling persists the AI's chosen value enum-validated. It also tightens
the advisory detector (now advisory-quality, not correctness-critical) so the hint is not
egregiously wrong on continuous features, and pins a titanic-shaped regression fixture.

Purpose: make COMP-01's "correct CV scheme" clause hold by making the scheme an AI decision, not
a data-derived default committed as trusted fact that every Phase-3+ cycle re-reads.
Output: cv.scheme becomes an enum-validated AI decision; cv-evidence.json's recommendation is
labeled advisory; the titanic false-positive is fixed and pinned; SKILL.md documents the real flow.
</objective>
