Safe-merge / deep-merge policy (D-02) — the exact rules the tests pin:
```
files (docs, .env, .gitignore, pyproject.toml):  create-if-absent; never overwrite.
  EXCEPTION — .claude/settings.json is NOT create-if-absent: it is DEEP-MERGED in 01-03 (D-09)
  (union allowedDomains + permissions.allow, preserve every other user key). 01-02 does NOT
  write settings.json at all — the settings merge is owned entirely by 01-03. This line lists
  only the flat create-if-absent files this plan writes; settings.json is called out here purely
  so 01-02 and 01-03 do not state contradictory policies for the same file.
control-plane JSON (config.json, state.json):
  - if absent  -> write from template.
  - if present & valid JSON -> DEEP-MERGE: add any missing keys/subkeys from the template
                               (recursively, e.g. add cv.scheme if absent) WITHOUT touching
                               existing values; hand-edited nested values survive.
  - if present & MALFORMED (json.JSONDecodeError) -> DO NOT overwrite; print the offending
                               path + parse error to stderr and exit non-zero (fail-clear;
                               a corrupt file is preserved for user repair, never silently replaced).
  - a key that exists with a different TYPE than the template is left as the user's value
    (we only ADD missing keys; existing keys are never mutated by merge).
only `--set-execution-target` may overwrite an existing value, and only the execution_target key.
```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user CLI args -> filesystem | `--slug` / `--execution-target` become file content |
| existing workspace -> re-run | idempotent merge must not destroy user edits or clobber corrupt files |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-07 | Tampering | `--execution-target` / `--set-execution-target` | mitigate | argparse `choices` enum-validation; invalid value exits non-zero with no write (V5) |
| T-01-08 | Tampering | safe-merge re-run | mitigate | create-if-absent + JSON deep-merge (add-missing-only); only `--set-execution-target` may overwrite, and only that one key (D-02) |
| T-01-11 | Tampering | malformed control JSON on re-run | mitigate | Fail-clear: never overwrite an unparseable config.json/state.json; print path + parse error, exit non-zero, preserve bytes for user repair |
| T-01-12 | Tampering | scaffolding before user answers (D-01 bypass) | mitigate | Mechanical slug gate: a fresh workspace without `--slug` creates nothing and exits non-zero before any write |
| T-01-05 | Tampering | env mutation | accept | Scaffolder writes only inside `--workspace`; no global/home mutation; no installs in this plan |
</threat_model>
