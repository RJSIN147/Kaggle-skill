Script invocation contract (portability constraint — CLAUDE.md §Stack Patterns):
- Every script self-locates via `Path(__file__).resolve().parent` for template access.
- Every script takes `--workspace <dir>` (default `Path.cwd()`); NEVER relies on `${CLAUDE_SKILL_DIR}`/`${CLAUDE_PROJECT_DIR}`.
- `python3 scripts/init_workspace.py --workspace <dir>` — scaffold (guided values passed as flags: `--slug`, `--execution-target`).
- `python3 scripts/init_workspace.py --workspace <dir> --set-execution-target {local|kernel}` — change target later (SETUP-02).
- `python3 scripts/check_credentials.py --workspace <dir>` — detect/normalize/validate; writes `credentials` status into `<dir>/control/state.json`.

Control-plane schemas the tests assert (D-10):
```jsonc
// control/config.json
{ "workspace_version": 1, "competition_slug": "<slug>",
  "execution_target": "local", "cv": { "scheme": null }, "created": "<iso8601>" }
// control/state.json
{ "credentials": "UNVALIDATED", "next_exp_id": 1 }
// control/ledger.jsonl  -> empty file (append-only, one experiment per line later)
```
