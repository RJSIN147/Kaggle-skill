Script invocation contract (portability constraint — CLAUDE.md §Stack Patterns):
- Every script self-locates via `Path(__file__).resolve().parent` for template access.
- Every script takes `--workspace <dir>` (default `Path.cwd()`); NEVER relies on `${CLAUDE_SKILL_DIR}`/`${CLAUDE_PROJECT_DIR}`.
- `python3 scripts/init_workspace.py --workspace <dir> --slug <slug> --execution-target {local|kernel}` — scaffold. `--slug` is REQUIRED for a fresh workspace (mechanical D-01 gate — the script creates nothing without it).
- `python3 scripts/init_workspace.py --workspace <dir> --set-execution-target {local|kernel}` — change target later (SETUP-02).
- `python3 scripts/check_credentials.py --workspace <dir> [--yes]` — detect/normalize/validate; `--yes` carries the user's consent (from the SKILL's D-03 prompt) for chmod/.env-population fixes; writes `credentials` status into `<dir>/control/state.json`.

Because every script is invoked as `python3 scripts/...`, `SKILL.md` `allowed-tools` MUST include `Bash(python3 scripts/*)` so the documented invocation path is pre-approved (addresses REVIEWS.md invocation-path-consistency finding).
