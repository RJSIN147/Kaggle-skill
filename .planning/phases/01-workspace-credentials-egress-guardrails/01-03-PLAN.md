settings.json MERGE policy (D-09 — this is the HIGH-confirmed fix; NOT create-if-absent):
```
if .claude/settings.json is ABSENT   -> write the template.
if PRESENT & valid JSON              -> DEEP-MERGE:
   - ensure sandbox.enabled = true (set if missing/false);
   - UNION sandbox.network.allowedDomains with the required hosts (preserve existing entries,
     add missing ones, de-duplicate; existing extra domains are kept);
   - UNION permissions.allow with the required entries (preserve the user's existing allow list);
   - PRESERVE every other user key untouched (e.g. an unrelated "env" block).
if PRESENT & MALFORMED               -> DO NOT overwrite; print path + parse error, exit non-zero
   (same fail-clear policy as the control-plane JSON in 01-02).
```
Wildcard note: `*.storage.googleapis.com` is included per the Claude Code settings schema (research-cited); the automated test asserts EXACT required-host membership (not wildcard parsing). Runtime wildcard/enforcement behavior is verified by the Task 3 checkpoint, not by the unit test.

Leak-guard (D-15) — the hook body is `scripts/leak_scan.py` copied to `<workspace>/.githooks/pre-commit` (chmod +x), wired via `git config core.hooksPath .githooks`. It scans staged CONTENT (not only unified-diff lines): enumerate staged files with `git diff --cached --name-only --diff-filter=ACM`, then for each read the staged blob via `git show :<path>` and scan the full text. Patterns (stdlib `re`, case-insensitive where noted): `kag(a|r)t_...`, `KGAT_...`, dotenv assignments `(?i)(export\s+)?KAGGLE_(KEY|API_TOKEN)\s*=\s*['"]?\S`, and `"key"\s*:\s*"[0-9a-f]{32}"` (32-hex ONLY inside the JSON `"key":` context to avoid false-positives on commit SHAs / unrelated hashes). Any hit -> print `[BLOCKED] ...` (pattern names only, never the secret) to stderr, exit 1. Documented, non-normalizing override: `git commit --no-verify` (documented in references/egress-allowlist.md as the escape hatch; the hook itself never bakes in a bypass).

git init (portability): prefer `git init -b main`; if that fails (older git without `-b`), fall back to `git init` then set the branch to `main` (`git symbolic-ref HEAD refs/heads/main` before the first commit, or `git branch -m main` after it).

Initial commit (idempotent, scaffold-scoped — the second HIGH-confirmed fix): stage ONLY the scaffold-owned manifest paths that exist and are not gitignored — `control/config.json`, `control/state.json`, `control/ledger.jsonl`, `competition.md`, `strategy.md`, `README.md`, `.gitignore`, `.claude/settings.json`, `pyproject.toml`, `.githooks/pre-commit` — via `git add -- <path> ...`. NEVER `git add -A`. Then commit `chore: scaffold workspace` ONLY if (a) there are staged changes and (b) no prior `chore: scaffold workspace` commit already exists (re-run must not create a second scaffold commit).
