CRITICAL (01-RESEARCH.md §Summary / §Pitfall 1): egress for the `kaggle`/`uv`/`git` CLIs is enforced ONLY by `sandbox.network.allowedDomains`. `permissions.allow: WebFetch(domain:...)` governs Claude's own WebFetch tool and does NOT constrain the CLI. The shepsci exemplar uses only WebFetch permissions (and a stale `api.kaggle.com`) — do NOT copy it. Write the sandbox block as the real control; keep WebFetch rules as a documented second layer.

settings.json shape (D-08 hosts — GCS backend mandatory or `kaggle competitions download` silently breaks in Phase 2):
```jsonc
{
  "sandbox": { "enabled": true, "network": { "allowedDomains": [
    "www.kaggle.com", "kaggle.com", "storage.googleapis.com", "*.storage.googleapis.com",
    "pypi.org", "files.pythonhosted.org",
    "github.com", "raw.githubusercontent.com", "codeload.github.com",
    "repo.anaconda.com", "conda.anaconda.org" ] } },
  "permissions": { "allow": [
    "WebFetch(domain:www.kaggle.com)", "Bash(kaggle *)", "Bash(uv run *)", "Bash(git *)" ] }
}
```
Deliberately EXCLUDED: Hugging Face / model CDNs (Phase 4, per Deferred Ideas).

Leak-guard (D-15) — the hook body is `scripts/leak_scan.py` copied to `<workspace>/.githooks/pre-commit` (chmod +x), wired via `git config core.hooksPath .githooks`. Scans `git diff --cached --unified=0` for: `kag(a|r)t_...`, `KGAT_...`, `KAGGLE_(KEY|API_TOKEN)=<value>`, and `"key":"<32-hex>"` (32-hex only inside the JSON `"key":` context to avoid false-positives on commit SHAs). Any hit -> print `[BLOCKED] ...` to stderr, exit 1.
